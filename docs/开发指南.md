# 开发指南

面向想改代码、加功能的人（包括未来的自己）。

## 总体架构

```
┌─────────────────────────────────────────────┐
│  ui/  前端（原生 JS，无框架无构建）           │
│  index.html + app.js + style.css            │
│  通过 window.__TAURI__.core.invoke 调命令    │
├─────────────────────────────────────────────┤
│  src-tauri/  Tauri 2 壳                     │
│  lib.rs     命令层（薄封装，只做参数搬运）    │
│  android.rs 安卓原生能力（JNI：分享/相册/打开）│
├─────────────────────────────────────────────┤
│  core/  scanking-core（全部重逻辑，纯 Rust）  │
│  detect    文档边缘检测                      │
│  geometry  四边形几何 + 透视矫正             │
│  filters   增强滤镜                          │
│  pdf       PDF 生成（手写，零依赖）           │
│  idcard    证件拼页                          │
│  store     文档库存储（JSON + 文件目录）      │
│  ocr       离线 OCR（tract + PaddleOCR v4）  │
└─────────────────────────────────────────────┘
```

设计原则：**core 不知道 Tauri 的存在**，可独立测试、独立复用；命令层不写业务逻辑。

## 数据流（一张照片的一生）

1. 前端拿到图片（相机 canvas / 文件选择）→ base64 → `page_import`
2. `store.add_page`：解码 → EXIF 转正 → 限长边 3200px → 存 `orig.jpg` + `preview.jpg`（1600px，编辑器用）
3. `detect::detect_quad`：缩到 600px → 灰度 → 高斯模糊 → 自适应 Canny → 膨胀 → 轮廓 → 凸包 → DP 简化为四边形（失败退化为最小外接矩形，再不行返回整图）
4. `geometry::warp_quad`：四角求单应矩阵 → Catmull-Rom 双三次重采样 → `warped.jpg`（缓存，换滤镜不用重矫正）
5. `filters::apply_filter` → `work.jpg`（最终效果）+ `thumb.jpg`
6. 导出：全部 work.jpg 以 DCTDecode 原样嵌入 PDF（`pdf.rs` 手写 xref，无重编码）
7. OCR：work.jpg → det（960 定长）→ 连通域 → 最小外接矩形 unclip → 逐框 warp 切条 → cls 转正 → rec（宽度分桶）→ CTC 解码 → 阅读顺序排序

## 存储布局

```
<app_data>/library/
  docs/<doc_id>/meta.json            文档元数据（标题/标签/页序）
  docs/<doc_id>/pages/<page_id>/
    orig.jpg      原图（已转正、限分辨率）
    preview.jpg   编辑器预览（1600px）
    warped.jpg    矫正后未加滤镜（缓存）
    work.jpg      最终效果
    thumb.jpg     缩略图
    ocr.txt       识别文本
  exports/*.pdf
  models/         OCR 模型（首启从内置资源安装）
```

## 命令 API（lib.rs）

| 命令 | 参数（JS 侧 camelCase） | 返回 |
|---|---|---|
| lib_list / lib_search | query? | DocSummary[] |
| doc_create | title | DocMeta |
| doc_get | docId | { meta, pages: PageInfo[] } |
| doc_rename / doc_set_tags / doc_delete | docId, ... | () |
| page_import | docId, data(base64) | PageInfo |
| page_redetect | docId, pageId | quad |
| page_set_geometry | docId, pageId, quad?, rotation | PageInfo |
| page_set_filter | docId, pageId, filter | PageInfo |
| page_delete / pages_reorder | docId, ... | () |
| idcard_compose | docId, pageA, pageB | PageInfo |
| doc_export_pdf | docId, mode(fit/a4), quality | 路径 |
| share_file / open_file | path（必须在数据目录内） | () |
| save_page_gallery | docId, pageId | 保存位置 |
| page_ocr / doc_ocr_text | docId, pageId? | 文本 |
| ocr_ready / install_model | name, data, append | bool / 字节数 |

约定：Rust 端参数 snake_case，JS 端自动转 camelCase；图片一律 base64 字符串；返回的文件路径用 `convertFileSrc(path) + "?v=" + bust` 显示。

## 开发循环

```
cargo tauri dev                 # 桌面窗口秒开，全功能可调（最快迭代路径）
cargo test -p scanking-core     # 核心库测试（检测精度/矫正/滤镜/PDF/存储）
.\build.bat                     # 一键 Android debug APK（含全部补丁）
.\build.bat -Release            # 正式版（需签名，见 README）
```

OCR 端到端测试：`powershell -ExecutionPolicy Bypass -File scripts\ocr_test.ps1`（桌面渲染测试图跑真模型）。

## 加功能的姿势（示例：新滤镜）

1. `core/src/filters.rs`：枚举加变体 + `from_name/name` + 实现函数
2. `core/tests/pipeline.rs`：test_filters_run 的循环里补一项
3. `ui/index.html`：filter-bar 加一个 chip（data-filter 与 name 对齐）
4. 完事 —— 命令层和存储不用动（滤镜名是透传的字符串）

新命令：core 加方法 → lib.rs 加 `#[tauri::command]` 薄封装 → `generate_handler!` 列表注册 → JS `call("名字", {...})`。耗时操作包 `spawn_blocking`。
