# 项目记忆

给"接手的人"看的上下文快照 —— 无论是几个月后的自己、新协作者，还是 AI 助手。读完这一篇就能接着干活。

## 一句话定位

对标扫描全能王核心体验的**免费、无广告、完全离线**扫描 App。起因：作者被商业扫描 App 的收费和广告惹毛了（2026-07-17 立项）。数据永远只在用户设备上，这是产品底线，不可妥协。

## 关键技术决策及理由

| 决策 | 理由 |
|---|---|
| Tauri 2 而非原生安卓 | 一套代码桌面可调试（迭代快 10 倍）+ Android 出包；作者主语言 Rust |
| 图像核心纯 Rust 自研（无 OpenCV） | 交叉编译零原生依赖；检测/矫正/滤镜算法量可控（各百来行） |
| OCR 用 tract 而非 onnxruntime | 纯 Rust，不用往 APK 塞 .so、不用折腾 jniLibs；速度可接受（页级秒等待） |
| 模型 PaddleOCR v4（从 rapidocr PyPI wheel 提取） | 中英文效果好、Apache-2.0、拿取稳定 |
| PDF 手写生成器 | JPEG 直嵌 DCTDecode 仅 ~150 行，避开 printpdf 版本动荡 |
| 存储用 JSON+目录而非 SQLite | 少一个原生依赖，文档量级下够用 |
| 图片走 base64 IPC 而非 raw body | 求稳（raw body API 当时无法验证）；大文件分 2MB 块 |
| 前端零框架零构建 | 用户机器不用装 Node；整个前端 3 个文件 |
| 安卓分享/相册用 JNI 直调 | opener 插件在安卓有 bug；自研 4 个函数覆盖分享/打开/存相册 |
| 复用 Tauri 的 FileProvider | 同类名 provider 只能有一个实例生效（血泪，见避坑手册） |

## 当前状态（2026-07-19）

**真机已验证**：拍摄/相册导入、自动裁边、透视矫正、五种滤镜、多页管理、PDF 导出与打开、微信分享（图片+PDF）、保存相册、离线 OCR 中英文、证件拼页、全文搜索、深色模式、Clean UI。

**桌面已验证**：核心库 10 项测试全绿；OCR 端到端（Hello World 123 三行全对，置信度 0.99+）；PDF 经 pypdf/pdfium 双验证。

**构建**：`build.bat` 一键出包（自动补全部安卓补丁，幂等可重跑）；debug APK ≈ 60MB。

## 已知限制

- OCR 纯 CPU：手机整页 5-15 秒；识别行宽压到 640px 桶，超长行会被压缩影响精度
- 全文搜索是线性扫描 ocr.txt，文档几千份后要上索引
- 保存相册需 Android 10+（MediaStore relative_path）
- 矫正后文字方向不自动判断（90° 侧转需手点旋转；cls 模型只管 180°）
- 相机预览依赖 WebView getUserMedia，个别 ROM 不放行（有系统相机/相册兜底）
- PDF 无文字层（不可搜索），OCR 文本单独存

## 路线图（按价值排序）

1. 正式版签名发布流（脚本已支持 -Release + keystore.properties 自动接线）
2. 可搜索 PDF：不可见文字层，需嵌入 CJK 字体子集（工程量最大的一块）
3. OCR 后按文字方向自动转正页面（cls 已内置，扩展到 90°/270° 需检测行方向）
4. 相机实时边缘预览（JS 端轻量检测或 wasm 化 detect）
5. 模型改为可选下载包，APK 瘦到 ~25MB
6. 文档加密/应用锁；多语言 UI

## 文件索引

- 核心算法：`core/src/{detect,geometry,filters,pdf,idcard,store,ocr}.rs`
- 命令层：`src-tauri/src/lib.rs`；安卓 JNI：`src-tauri/src/android.rs`
- 前端：`ui/{index.html,app.js,style.css}`
- 构建：`build.bat` → `scripts/build_apk.ps1`（所有安卓补丁的唯一真源）
- 模型：`scripts/fetch_models.ps1` → `ui/models/`（onnx 不入库，keys.txt 入库）
- 测试：`core/tests/{pipeline,ocr_e2e}.rs`、`scripts/ocr_test.ps1`
- 文档：`docs/{开发指南,避坑手册,美术规范,项目记忆}.md`

## 行事风格约定

- 出问题先复现、拿到**原始错误文本**再动手；错误信息要把底层原文带出来（JNI 的 Java 异常 toString 就是这么救回来的）
- 所有安卓工程修改必须进 `build_apk.ps1` 做成幂等补丁，gen 目录随时可删可重建
- 提交信息用中文，说清"修了什么 + 为什么"
